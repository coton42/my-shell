# my-shell

C言語で実装した POSIX 準拠の Linux 用シェルです。パイプライン、I/O リダイレクト、ジョブ制御などの基本機能を備えています。

## 特徴

- **パイプライン** — 複数コマンドをパイプ (`|`) で連結して実行
- **I/O リダイレクト** — 標準入力・標準出力・標準エラー出力をファイルへリダイレクト
- **バックグラウンド実行** — `&` でコマンドをバックグラウンドで実行
- **ジョブ制御** — `jobs`、`fg`、`bg` によるジョブの管理
- **組み込みコマンド** — `cd`、`pwd`、`exit` など
- **PATH 解決** — 環境変数 `$PATH` を使ってコマンドを自動検索

## 動作要件

- Linux (POSIX 準拠環境)
- GCC または Clang (C11 対応)
- GNU Make

## ビルド

```sh
make
```

生成されるバイナリ: `./my-shell`

ビルド成果物の削除:

```sh
make clean
```

## 起動方法

```sh
./my-shell
```

起動するとプロンプト `$ ` が表示され、コマンド入力待ちになります。

## コマンドと構文

### 組み込みコマンド

| コマンド | 説明 |
|---|---|
| `cd [path]` | ディレクトリを移動。`path` 省略時は `$HOME` へ移動 |
| `pwd` | カレントディレクトリを表示 |
| `exit` | シェルを終了 |
| `jobs` | バックグラウンドジョブの一覧を表示 |
| `fg [%id]` | 指定したジョブをフォアグラウンドで再開 |
| `bg [%id]` | 停止中のジョブをバックグラウンドで再開 |

### パイプライン

コマンドを `|` で繋ぐことで、前のコマンドの標準出力を次のコマンドの標準入力に渡します。

```sh
$ cat file.txt | grep keyword | wc -l
```

### I/O リダイレクト

| 記法 | 説明 |
|---|---|
| `< file` | `file` を標準入力として使用 |
| `> file` | 標準出力を `file` に書き出し（上書き） |
| `2> file` | 標準エラー出力を `file` に書き出し |

```sh
$ sort < input.txt > output.txt
$ cmd 2> error.log
```

### バックグラウンド実行

コマンド末尾に `&` を付けるとバックグラウンドで実行されます。

```sh
$ sleep 60 &
```

## 使用例

```sh
# 基本的なコマンド実行
$ ls -la

# パイプライン
$ ls /usr/bin | grep ssh | sort

# ファイルへのリダイレクト
$ echo "hello" > greet.txt
$ cat < greet.txt

# バックグラウンドジョブ
$ sleep 30 &
$ jobs
[1] running   sleep 30

# フォアグラウンドに戻す
$ fg %1

# ディレクトリ移動
$ cd /tmp
$ pwd
/tmp
$ cd
$ pwd
/home/user
```
